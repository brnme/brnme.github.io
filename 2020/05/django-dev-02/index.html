留言板02 urls

半樵 创建于 2020-05-11 更新于 2020-05-12

留言板这个应用只需要一个输入页面，一个显示页面。

对应的链接就是/liuyan、/xianshi。

也就是说，两个url，两个view函数。

之前我鼓捣着玩的时候从最简单的HTTPResponse开始做过一回，所以这次直接弄内置视图函数了。

有种被新手教程坑了的感觉。

新手教学建议我把app放在manage.py同一目录下。以我错误的理解，这似乎会导致urls.py找不到board模块——事实证明我错得很离谱。

include函数的输入是字符串！

我还以为这里出问题了担心得要死。

记录一下我认为的路由过程。

先在主模块里查询urls，分出一个目录，把剩下的部分分包给子模块，让子模块去根据url确定具体的views函数。

include(< char >) 函数的输入是模块名.urls

接下来把model确定一下。

因为要用到通用视图，所以先确定两个页面的对应通用视图是什么。

xianshi => detailview

liuyan => formview
